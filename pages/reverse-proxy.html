Reverse Proxy

A dedicated app-server can handle more requests but ultimately still has physical limits. Generally, the more concurrent requests a server has the slower each request will be.

If one app-server can handle 1,000 requests, then two app-servers can handle 2,000 requests. It's possible to add additional app-servers to share the request load.

To do this we first need to add a reverse-proxy server that can delegate or proxy the requests across multiple app-servers.

The reverse-proxy is inserted before our app-server so requests go directly to the reverse-proxy. The reverse-proxy runs its own web-server software but it is configured to route requests to other servers.
